# Audio-Text Processing Platform

A local, high-performance Speech-to-Text (STT), Text-to-Speech (TTS), and Natural Language Processing (NLP) evaluation and processing platform. 

This repository contains a modular FastAPI backend, a React frontend dashboard, local model runtime wrapper services, and a comprehensive suite of benchmarking and evaluation tests.

---

## 1. Directory Structure

The project has been refactored into modular components:

```text
audio-text-platform/
│
├── backend/                       # Modular FastAPI Application
│   ├── routers/                   # Endpoint routers (health, survey, services)
│   ├── config.py                  # Folder paths and settings configurations
│   ├── database.py                # Redis client connection manager
│   ├── helpers.py                 # FFmpeg transcoder and mathematical stats helpers
│   ├── models.py                  # Pydantic request/response validator schemas
│   ├── main.py                    # Server lifespan (warm-loading) and entrypoint
│   └── requirements.txt           # Python application packages
│
├── frontend/                      # React Frontend Application (Vite + Chart.js)
│   ├── src/
│   │   ├── App.jsx                # Main Survey Tasks & Admin Stats Dashboard
│   │   └── main.jsx & index.css   # Main app launcher and styles

├── services/                      # Core AI Pipeline Model Loaders
│   ├── nlp/nlu_service.py         # spaCy Intent parsing, NER, and Sentiment scoring
```

## 2. Quick Start (Running the App)

### Prerequisites
* **Docker Desktop** must be running.
* **Node.js & npm** must be installed.

### Step 1: Start the Backend (Docker)
Build and start the FastAPI server and Redis cache in the background:
```bash
docker compose up --build -d
```
* Note: On the first boot, the server warm-loads the Whisper and Piper models concurrently, which might take up to a minute to download and cache weights. Verify status at `http://localhost:8000/api/health`.

### Step 2: Start the Frontend (Vite)
Install package dependencies and start the local development server:
```bash
# Navigate to the frontend directory
cd frontend

# Install node dependencies
npm install

# Start Vite with local network exposure enabled
npm run dev -- --host
```

## 3. Local Network Sharing (Human Testing)
To allow other devices (like smartphones, tablets, or other laptops) on the same Wi-Fi network to participate in the MOS evaluation survey:
1. Identify your host Mac's local IP address (e.g., `192.168.31.93`).
2. Share the exposed network URL printed by Vite:
   ```text
   http://<your-mac-ip>:5173
   ```
3. The frontend is configured to route API calls dynamically to `http://<your-mac-ip>:8000` so that submissions are successfully recorded in your central Docker database.

## 4. Running Automated Tests & Benchmarks

### 🧪 API & Route Verification
To run the full integration test suite, execute the following command in your terminal:
```bash
./.venv/bin/python -m unittest tests/test_api_transcribe.py tests/test_api_nlu.py tests/test_api_synthesize.py tests/test_api_process.py
```

### 🧪 Memory Leak Check
To verify RAM usage stability over 100 consecutive requests under load, run:
```bash
./.venv/bin/python -m unittest tests/test_memory_leak.py
```

### 📊 Simulated Survey Population (MOS Survey Round 2)
To instantly populate the Admin Dashboard with 20 realistic survey submissions (averaging an overall MOS rating $>3.5$), run the simulation utility:
```bash
./.venv/bin/python tests/generate_mock_responses.py
```
Check the live metrics by navigating to the **Admin / Stats Dashboard** tab on the web client interface.
